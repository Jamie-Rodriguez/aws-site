Learning to Play Blackjack through Deep Reinforcement Learning

https://github.com/Jamie-Rodriguez/RL-Blackjack

Using OpenAI Gym for the environment, I created an agent that can learn to play blackjack with the most optimal strategy through deep reinforcement learning (RL).

Before moving to full function approximation, I first implemented a tabular method of Monte Carlo sampling. This was to test and ensure that the RL algorithm was functioning correctly, before adding a neural network on top.

Once I was satisfied with the RL algorithm, it was time to replace the state-action table with a function approximator - in this case a neural network.

The deep reinforcement learning agent was able to converge to the optimal solution, however it took much longer than simpler tabular method. This is an expected cost of using function approximation over tabular methods however.

See my GitHub link for more in-depth explanation and analysis.

Tic-Tac-Clojure

https://github.com/Jamie-Rodriguez/tic-tac-clojure

Implemented a tic-tac-toe game engine in Clojure, in a pure functional style, using a modern Clojure stack (CLI + Deps).

I created this for two reasons:

  1. To learn modern Clojure development, specifically using CLI + Deps with unit testing.
  2. To use this engine/environment in the future to experiment with reinforcement learning agents.

Microservice Web-API

https://github.com/Jamie-Rodriguez/microservice-web-api

I hope that this repository can serve as an easy to understand, minimal example for "containerising" (and orchestrating) a microservice-style web server application.

Equally important is the example of a modern Clojure webserver - as all the resources I could find were using much older, deprecated libraries. I hope this can be a useful reference for others.

Eventually I hope for this example to be as close to a production-ready system as possible i.e. it will have containerisation, orchestration, scalability and observability to ensure as much reliability as possible. But that's enough buzzwords for now.

Perhaps way in the future I will also add capability to deploy this system to a cloud environment, probably using a platform-agnostic tool such as Terraform.

C: Command Line Arguments Processing

https://github.com/Jamie-Rodriguez/C_Command_Line_Args_Processing

getopt.h is the most popular library used to process command line arguments in C. However I found it a bit cumbersome to use as well as not as flexible as required for my uses. As well as this, it is not a standard library and so is not guaranteed to be present on all systems.

My technique is pretty simple, I use a hash function to compare the command line flags inside a switch statement. I find this cleaner and safer than using something like strcmp() in this specific case.

The GitHub repo contains two code examples to inspect getopt.h vs my hash method.

As I write this, I think I should be able to use C macros to automatically hash the command line flags at the preprocessor stage, rather than manually as it is done currently. If I revisit this in the future I will add this in.
